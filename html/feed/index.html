Alex Lande 2012-03-23T20:03:41-06:00 http://www.alexlande.com/ Alex Lande alexlande@gmail.com Conditional Stylesheets Revisited 2012-01-20T00:00:00-07:00 http://www.alexlande.com/articles/conditional-stylesheets <p>If you&#8217;re writing CSS in 2012, there&#8217;s a good chance that you&#8217;ve tried pre-processing through SASS or LESS. <a href='http://nicolasgallagher.com/'>Nicolas Gallagher</a> recently described a SASS technique for mobile first development that allows you to easily deliver large screen styles to outdated browsers. After experimenting with the technique, I&#8217;ve found some ways to push it even further.</p> <h3 id='preprocessing_to_the_rescue'>Pre-processing to the Rescue</h3> <p><a href='http://nicolasgallagher.com/mobile-first-css-sass-and-ie/'>Nicolas&#8217; technique</a> uses SASS imports to create a media query-laden style sheet for capable browsers and an IE-specific style sheet that delivers the same styles in a conditional comment.</p> <p>It feels sort of odd to use an IE style sheet in a world of <a href='http://paulirish.com/2008/conditional-stylesheets-vs-css-hacks-answer-neither/'>conditional classes</a>, but if we&#8217;re going to we might as well make the most of it. Even if you aren&#8217;t much of a pre-processor, you&#8217;ve likely realized that modularizing your CSS in this way doesn&#8217;t have to stop with media queries. Since we&#8217;re already creating an IE-specific style sheet, we might as well import our IE-specific styles (written with the aforementioned conditional classes) into that file and leave them out of the other, like so:</p> <pre><code>@import &quot;480.scss&quot;; @import &quot;768.scss&quot;; @import &quot;992.scss&quot;; @import &quot;ie.scss&quot;;</code></pre> <p>Easy as that, your main styles are free of <code>.ie7</code> and <code>.oldie</code>. Feels good, right?</p> <h3 id='html5shiv_optimizations'>html5shiv Optimizations</h3> <p>One of the little-discussed benefits of media query polyfills like <a href='https://github.com/scottjehl/Respond'>Respond.js</a> is that they make excellent companions to html5shiv. If you&#8217;re using both, and a user visits your site on a legacy browser without JavaScript, they&#8217;ll only receive your base styles. If you&#8217;ve planned for this, that means they&#8217;ll see a functional but basic page, one that isn&#8217;t crippled by the lack of styled HTML5 elements and media queried layout.</p> <p>The same thing is possible with the SASS importing method. It&#8217;s easiest if you&#8217;re using <a href='https://github.com/Modernizr/Modernizr'>Modernizr</a>, which has both html5shiv and a conditional loader called yepnope built in. Using yepnope, we can ensure that the legacy style sheet only loads when html5shiv is in use:</p> <pre><code>&lt;!--[if (lt IE 9) &amp; (!IEMobile)]&gt; &lt;script&gt;yepnope(&#39;legacy.css&#39;);&lt;/script&gt; &lt;![endif]--&gt;</code></pre> <p>You can toss a script like <a href='https://github.com/keithclark/selectivizr'>Selectivizr</a> in at the same time, if you&#8217;re so inclined.</p> <h3 id='be_not_afraid'>Be Not Afraid</h3> <p>Although I&#8217;ve been talking about SASS here, this style of modular CSS isn&#8217;t dependent on it. It <a href='https://gist.github.com/1407227'>works with LESS</a>, with a build script a la HTML5 Boilerplate, and if you&#8217;re really masochistic you could do it by hand with copy and paste. (Don&#8217;t do that.)</p> <p>If your design is fully responsive and is built on a fluid grid, IE users will still enjoy an experience that meets them halfway.</p> <p>My only concern is that the technique is IE specific. In the last week, multiple people have visited this site using Firefox 3. That&#8217;s pretty strange, but it&#8217;s worth noting that there are still people using legacy browsers other than IE. As always, take a look at your statistics before you do anything rash.</p> Daybreak 2011-11-04T00:00:00-06:00 http://www.alexlande.com/articles/daybreak <p>This is a rough draft.</p> <p>This is my third rough draft, specifically.
